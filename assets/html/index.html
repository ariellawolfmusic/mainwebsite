Core Features

Coding in Seconds

Use Mimosa's command line to build out a skeleton app that is ready to go with transpilers, pre-processors, and a server if you need one. Or use a pre-built skeleton or example app already equipped with your favoriate libraries.

Watch And Serve Assets

Mimosa will keep track of your assets and when you save something kick off all sorts of awesome. Mimosa will also optionally serve assets for you or invoke a node server so that you only need a single process running during development.

Convention + Config

Sane defaults keep you coding instead of figuring out config. If you stick close to Mimosa's conventions you may never look at or update the configuration. But Mimosa is heavily configurable if you find yourself needing to vary away from its conventions.

Concise Command Line

Mimosa has just a few commands to master and you won't need to invent your own. Mimosa's commands have flags to engage when you want perform common tasks like minify, concatenate, or package your application.

Modules

Utility & Library Integrations

Live reload, Bower, jshint, eslint, ember.js, and the list goes on. Building a module is simple, so if the one you deeply need doesn't exist, bridging the gap is a piece of cake.

Compilers

SASS, Less, Stylus, CoffeeScript, 6to5, Handlebars, Jade and dozens of others. Mimosa's core provides the heavy lifting for compilers making writing a compiler module easy, so there plenty from which to chose.

Testing

Karma, Testem, QUnit, and Mocha are a few of the testing modules out there, some with deep intergrations with libraries like Ember.

Module Loading

Mimosa's AMD/require.js support is second to none, but if commonjs & browserify are your thing, there are modules for that too.
