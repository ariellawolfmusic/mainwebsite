Installation it's a piece of cake

1) node.js

Mimosa is written in node.js and that needs to be installed. Download and install the latest stable version over 0.8.0. (Do not use the unstable version of node.js, currently 0.11.0.) The node.js install comes with the Node Package Manager (npm).

2) mimosa

Use npm to install Mimosa.

$ npm install -g mimosa

The -g flag installs Mimosa globally so it can be run from the command line anywhere. You may need to use sudo to install this depending on your system's setup.

Where to Start with Development?

You've got four choices, all detailed below:

  • Start by cloning/downloading a demo application if you want something meaty.
  • Use one of Mimosa's application skeletons if you just want a little boilerplate, some client libraries and some sample code.
  • Start with a fresh Mimosa generated scaffold.
  • Hook up your existing application.

Demo Applications great way to learn Mimosa

Before creating your own app, play around with someone else's! Download an app from GitHub and use Mimosa's command line interface to play with the code and get comfortable with Mimosa and the technologies inside the demo apps.

As more demo apps are built, they'll get added here. If you put a little something together using Mimosa let us know so we can add it to the list! Either open an issue or shoot a message to @mimosajs.

  • ReactBackbone A Backbone/React with require.js and bower support.

  • Testem A small CoffeeScript, Stylus, Handlebars app will a full Mimosa-managed Testem/Mocha/Sinon/Chai test suite.

  • AngularFun A mid-sized CoffeeScript, LESS, Stylus, Angular.js, Bootstrap application.

  • AngularFunCommonJS Same as the AngularFun project, but utilizing a mixture of both AMD and CommonJS code.

  • EmberEmblem A small Ember.js project utilizing Emblem for templating. Also: CoffeeScript, Jade, Express.

  • Ember The Mimosa starter app but Ember-ized. CoffeeScript, Emblem, SASS, Jade, Express.

  • EmberPeepcode The Peepcode Ember example. Bower, Testem, Ember.js, Emblem, CoffeeScript, Foundation, Stylus, Express and Jade.

  • EmberCommonJS A small (but growing) Ember example using browserify and CommonJS. Bower, Testem, Ember.js, Handlebars, CoffeeScript, Foundation, Express and Jade.

  • BrowserifyExample The Mimosa starter app but browserify-ified. Browserify, Bower, CoffeeScript, Dust, Stylus, Express and Jade.

  • DynamicRequire A quick little example showing how to use Mimosa to build a requirejs app with dynamic module loading.

Pre-Built Skeletons execute "mimosa skel:list" and see what's available

A skeleton is a small application that utilizes and configures a group of components. Some of the Demo Applications above are also skeletons if they are small enough to allow you to get started without first removing 50 files to get down to something bare and essential.

Mimosa maintains a "skeleton registry" that you can pull from to start your application. From the command line execute mimosa skel:list to get a list of pre-built skeletons. Follow the instructions to install one of your choosing!

Mimosa needs more skeletons! Check out the documentation and whip something up real quick using the technologies you love so that others can benefit.

Quick Start - New App you'll be coding soon

If you want to build an app from scratch, or just play around with Mimosa, start here.

1) Create a new application

Inside the desired folder, create an app by executing mimosa new.

$ mimosa new nameOfApplicationHere

Follow the prompts and choose the desired transpilers/pre-processors, server, and server view technologies. If you aren't sure which ones to choose, that's perfect. Mimosa is a great tool to learn them. For now go with the ones marked with an asterisk as they are Mimosa favorites.

2) Start the server

Change into the directory that was created and execute:

$ mimosa watch --server

Navigate to http://localhost:3000 in a browser to see the sample app. Mimosa will watch the assets directory of the project and compile changed files to the public directory. The web app serves assets in the public directory.

3) Start Coding

All set! To get the project started, Mimosa has delivered a bare bones web app with some code using the chosen technologies. Head over to the Commands tab to learn how to interact with Mimosa. If you want to play around with something meatier than the codebase that mimosa new delivers, try out a demo app.

Quick Start - Existing App easy to wire up

If you want to use Mimosa with an existing app, this is the place to start.

1) Get the config

From the command line, type mimosa config and Mimosa will generate a mimosa-config.js file in the current directory. This file controls everything Mimosa does. For now its contents will be relatively brief. mimosa config also delivers a mimosa-config-documented.coffee file which contains documentation for all the various configuration options.

2) Tweak the config

Place the mimosa-config at the root of a project. Either change the project's codebase to match Mimosa's defaults, or change the mimosa-config to work with the project. All of the configuration options are documented on the site as well as in the mimosa-config-documented.coffee.

The most important settings in the config, to get you started, are inside the top watch setting. Make sure that watch.sourceDir, watch.compiledDir, and watch.javascriptDir all point to the right place in the project.

3) Fire it up

Execute mimosa watch and Mimosa will build and watch project assets. Mimosa will generate errors if something is wrong with the config.
